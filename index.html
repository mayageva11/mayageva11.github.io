03 — Flagship project

🐕 FlakeHound

GitHub Marketplace npm published MIT licensed
Live Dashboard → CLI Repository npm Package Demo Repo

The problem it solves

A flaky test passes and fails on the same code — and every red build it causes trains your team to click "re-run" instead of investigating. Existing tools count pass/fail rates and tell you that a test is flaky; paid services like BuildPulse and Trunk.io charge for exactly that.

FlakeHound is free, open-source, and goes further: it normalises stack traces and clusters failures by their underlying cause — turning "23 failures over 3 weeks" into "4 unique bugs". It knows that a test failing 100% of the time since one commit isn't flaky, it's broken — and it gates CI on new regressions only, so the build fails once when a bug lands, not on every run until someone fixes it.

See it run

zsh — flakehound

How it works

🔍

Root-cause clustering

Jaccard similarity + greedy clustering groups structurally identical failures. "23 failures = 4 bugs" — one cluster, one bug, even when it spans many tests and weeks of history.

🤖

AI hypothesis layer

Claude API or local Ollama (privacy-first, zero cost) annotates each cluster with a one-line root-cause hypothesis. Swappable provider interface — the deterministic core works identically without it.

🚧

Quarantine subcommand

Auto-isolates high-confidence flaky tests via surgical AST edits (ts-morph). Files one GitHub issue per test. Auto-releases after N consecutive clean runs — no manual cleanup needed.

📊

Self-contained dashboard

One HTML file with the full report embedded — open from disk, attach as a CI artifact, or publish directly to GitHub Pages. No server, no account, no dependencies.
